Settings & Configuration Files

▼

Configuration File Hierarchy

Claude Code uses multiple configuration files with a specific hierarchy. Settings are applied from most general to most specific:

File Scope Description
~/.claude.json Legacy/Global Legacy file - often the only place certain settings work. Contains global settings, project configs, prompt history, and stats.
~/.claude/settings.json User (Global) Personal global settings that apply to every project. Preferred location for personal defaults like theme or model preferences.
.claude/settings.json Project Project-specific settings. Should be checked into source control for team-wide configurations.
.claude/settings.local.json Project Local Personal project tweaks. Ignored by Git, perfect for API keys or experimental settings.
~/.claude/mcp_servers.json User (MCP) Dedicated file for MCP server configurations.
.mcp.json Project (MCP) Project-level MCP configuration, designed to be checked into version control.
⚠️ Important: Any of these files can be overridden by enterprise policies or command-line arguments. Settings are applied with more specific files overriding more general ones.

Recommended Setup Structure

For reliability, use ~/.claude.json as shown in examples:

{

  "shiftEnterKeyBindingInstalled": true,

Key Configuration Settings

Permissions Object

Controls which tools and commands Claude Code can run:

  • allow: Tools that can run automatically without prompts
  • ask: Tools that require user confirmation first
  • deny: Tools that are completely blocked
Known Issue: "deny" rules are frequently ignored in practice, which poses a security risk. The AI might access files despite explicit deny instructions.

Allowed Tools Configuration

"allowedTools": [
  "Task",          // Task management
  "Bash",          // ⚠️ Dangerous: allows all system commands
  "Bash(git log:*)",  // Safer: only allows git log commands
  "Glob",          // Pattern matching
  "Grep",          // Text search
  "LS",            // List files
  "Read",          // Read files
  "Edit",          // Edit existing files
  "MultiEdit",     // Edit multiple files
  "Write",         // Create/write files
  "WebFetch",      // Fetch web content
  "WebSearch"      // Web searching
]

API Key Configuration

Multiple methods to set API key:

# Method 1: Environment variable (recommended)
export ANTHROPIC_API_KEY="your-api-key-here"

# Method 2: Add to shell profile
echo 'export ANTHROPIC_API_KEY="your-api-key-here"' >> ~/.bashrc
source ~/.bashrc

# Method 3: API Key Helper Script
echo 'echo ${ANTHROPIC_API_KEY}' > ~/.claude/anthropic_key_helper.sh
chmod +x ~/.claude/anthropic_key_helper.sh
claude config set --global apiKeyHelper ~/.claude/anthropic_key_helper.sh

CLI Reference - Commands & Flags

▼

Basic Commands

Command Description Example
claude Start interactive REPL claude
claude "query" Start REPL with initial prompt claude "explain this project"
claude -p "query" Query via SDK, then exit claude -p "explain this function"
cat file | claude -p "query" Process piped content cat logs.txt | claude -p "explain"
claude -c Continue most recent conversation claude -c
claude -c -p "query" Continue via SDK claude -c -p "Check for type errors"
claude -r "session-id" "query" Resume session by ID claude -r "abc123" "Finish this PR"
claude update Update to latest version claude update
claude mcp Configure MCP servers claude mcp add filesystem

Command Flags

--add-dir

Add additional working directories for Claude to access

# Add single directory
claude --add-dir ../apps

# Add multiple directories
claude --add-dir ../apps ../lib

# Use with print mode
claude --add-dir ../backend -p "Validate API calls"

--allowedTools

Specify tools that should be allowed without prompting

claude --allowedTools "Bash(git log:*)" "Bash(git diff:*)" "Read"

--disallowedTools

Specify tools that should be denied

claude --disallowedTools "Bash(git log:*)" "Edit"

--print, -p

Print mode - query and exit (SDK mode)

claude -p "query"

--append-system-prompt

Append custom instructions to system prompt (SDK mode only)

claude --append-system-prompt "Custom instruction"

--output-format

Set output format: text, json, or stream-json

claude -p "query" --output-format json

--input-format

Set input format: text or stream-json

claude -p --output-format json --input-format stream-json

--include-partial-messages

Include partial messages in stream-json output (SDK mode with stream-json only)

claude -p --output-format stream-json --include-partial-messages "query"

--verbose

Enable verbose logging

claude --verbose

--max-turns

Set maximum number of turns

claude -p --max-turns 3 "query"

--model

Specify model (sonnet, opus, or full model name)

claude --model claude-sonnet-4-20250514

--permission-mode

Set permission mode

claude --permission-mode plan

--permission-prompt-tool

Prompt for permission on specific tool

claude -p --permission-prompt-tool mcp_auth_tool "query"

--resume

Resume a specific session

claude --resume abc123 "query"

--continue

Continue most recent conversation

claude --continue

--dangerously-skip-permissions

Skip all permission checks (use with caution!)

claude --dangerously-skip-permissions

Switch Model During Session

/model

Available options in interactive menu:

  • Default (recommended): Opus 4.1 for up to 20% of usage limits, then use Sonnet 4
  • Opus: Opus 4.1 for complex tasks (reaches usage limits faster)
  • Sonnet: Sonnet 4 for daily use
  • Opus Plan Mode: Use Opus 4.1 in plan mode, Sonnet 4 otherwise (automatically switches between models)
Claude 3.5 Haiku Limitations:
  • Reduced reasoning capabilities
  • Limited context understanding
  • Simplified code analysis
  • Basic refactoring only
  • Limited framework knowledge
Recommended only for: Simple single-file edits, basic syntax corrections, quick code questions, learning Claude Code basics.

Model Context Protocol (MCP)

▼

MCP Overview

Model Context Protocol (MCP) allows Claude Code to connect to external tools and services, extending Claude's capabilities beyond its built-in features.

MCP Configuration Locations:
  • Project-specific: .claude/settings.local.json
  • User-specific local: ~/.claude/settings.local.json
  • User-specific global: ~/.claude/settings.json
  • Main Claude.json: ~/.claude.json (recommended)
  • Dedicated MCP file: ~/.claude/mcp_servers.json
  • Project shared: .mcp.json (for version control)

MCP Server Setup Commands

Basic Add Commands

# Add HTTP transport server
claude mcp add --transport http sentry https://mcp.sentry.dev/mcp

# Add SSE transport server
claude mcp add --transport sse asana https://mcp.asana.com/sse

# Add with environment variables
claude mcp add clickup --env CLICKUP_API_KEY=YOUR_KEY --env CLICKUP_TEAM_ID=YOUR_ID -- npx -y @hauptsache.net/clickup-mcp

# Add from JSON configuration
claude mcp add-json puppeteer "$MCP_JSON"

# List all MCP servers
claude mcp list

# Remove MCP server
claude mcp remove server_name

# Reset project choices
claude mcp reset-project-choices
Windows Note: NPX commands require cmd /c wrapper: 
claude mcp add myserver -- cmd /c npx -y server-package
Without this wrapper, you'll encounter "Connection closed" errors.

Scope Flags

Use --scope to specify where configuration is stored:

Scope Description Location
local (default) Available only to you in current project .claude/settings.local.json
project Shared with everyone via .mcp.json .mcp.json
user Available to you across all projects ~/.claude/settings.json 
# Add to user configuration
claude mcp add myserver --scope user -- npx -y server-package

MCP Configuration Structure

Example .mcp.json Structure

{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-filesystem", "/allowed/path"],
      "env": {
        "CUSTOM_VAR": "value"
      }
    },
    "memory": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-memory"]
    },
    "fetch": {
      "command": "npx",
      "args": ["-y", "@modelcontextprotocol/server-fetch"]
    },
    "remote-http": {
      "url": "https://api.example.com/mcp",
      "headers": {
        "Authorization": "Bearer ${API_TOKEN}"
      }
    },
    "remote-sse": {
      "url": "https://api.example.com/sse",
      "transport": "sse"
    }
  }
}

Configuration Fields

Field Description
command The server executable path
args Command-line arguments array
env Environment variables passed to server
url For SSE/HTTP server types
headers For SSE/HTTP server authentication
transport Transport type: "stdio", "sse", or "http"

Environment Variable Expansion

MCP configurations support environment variable expansion in .mcp.json files:

Supported Syntax

  • ${VAR} - Expands to the value of environment variable VAR
  • ${VAR:-default} - Expands to VAR if set, otherwise uses "default"
{
  "mcpServers": {
    "api-server": {
      "url": "https://api.example.com/mcp",
      "headers": {
        "Authorization": "Bearer ${API_TOKEN}",
        "Custom-Header": "${CUSTOM_VAR:-default-value}"
      }
    }
  }
}

MCP Troubleshooting

Common Issues

Connection Closed Errors (Windows):

If using NPX on Windows, you must use the cmd /c wrapper:

claude mcp add server -- cmd /c npx -y package-name
Timeout Issues:

Increase MCP timeouts via environment variables:

export MCP_TIMEOUT=30000        # 30 seconds for startup
export MCP_TOOL_TIMEOUT=60000   # 60 seconds for tool execution
Output Token Limits:

MCP responses are limited to prevent context overflow:

export MAX_MCP_OUTPUT_TOKENS=50000  # Default: 25000

Warning appears at 10,000 tokens.

Security Best Practices

Known Issues:
  • Deny Rules Often Ignored: "deny" permissions in the configuration are frequently not enforced, creating security risks
  • Unreliable Blocks: Files marked as denied may still be accessible by Claude
  • Workaround: Use filesystem permissions or place sensitive files outside allowed directories

Recommended Security Approach

  1. Use specific tool patterns: Always use Bash(command:*) patterns instead of general Bash
  2. Limit file access: Only add necessary directories with --add-dir
  3. Review permissions regularly: Use /permissions to audit and update
  4. Sensitive data isolation: Keep API keys, credentials, and sensitive files outside project directories
  5. Use .gitignore patterns: Claude respects .gitignore for some operations
  6. Monitor tool usage: Review what tools Claude is requesting

Common Pitfalls & Solutions

Issue: Settings not taking effect

Solution: Settings hierarchy can be confusing. Try setting in ~/.claude.json which often overrides other locations.

Issue: Deny rules ignored

Solution: Known bug. Use filesystem permissions or move sensitive files outside allowed directories.

Issue: MCP "Connection closed" on Windows

Solution: Use cmd /c wrapper for npx commands on Windows.

Issue: CLAUDE.md not being read from --add-dir

Solution: Known limitation. CLAUDE.md from additional directories may not auto-load. Copy important instructions to main project CLAUDE.md.

Issue: High API costs

Solution: Use Opus Plan Mode (auto-switches to Sonnet), or manually switch to Sonnet 4 for routine work. Reserve Opus for complex tasks.
